# Tauri v2 + SvelteKit

Desktop (and mobile) apps with a Rust backend and SvelteKit frontend. Native performance, web DX.

---

## Quick Reference

```yaml
Framework:   Tauri v2
Frontend:    SvelteKit (Bun + Svelte 5)
Backend:     Rust (Tokio-based Tauri runtime)
IPC:         invoke() / events
State:       tauri::State<T> (managed state)
Permissions: Capabilities system (src-tauri/capabilities/)
Build:       cargo tauri build
Dev:         cargo tauri dev
```

## Tauri Commands

Commands are the primary IPC mechanism — Rust functions callable from the frontend.

```rust
// src-tauri/src/commands/user.rs
use tauri::State;
use serde::{Deserialize, Serialize};

use crate::AppState;

#[derive(Debug, Serialize)]
pub struct User {
    pub id: String,
    pub name: String,
    pub email: String,
}

#[derive(Debug, Deserialize)]
pub struct CreateUserInput {
    pub name: String,
    pub email: String,
}

// Simple command — returns a value
#[tauri::command]
pub fn greet(name: &str) -> String {
    format!("Hello, {name}!")
}

// Async command with state access
#[tauri::command]
pub async fn get_users(
    state: State<'_, AppState>,
) -> Result<Vec<User>, String> {
    state.db
        .list_users()
        .await
        .map_err(|e| e.to_string())
}

// Command with both input and state
#[tauri::command]
pub async fn create_user(
    input: CreateUserInput,
    state: State<'_, AppState>,
) -> Result<User, String> {
    if input.email.is_empty() {
        return Err("email is required".to_string());
    }

    state.db
        .create_user(input.name, input.email)
        .await
        .map_err(|e| e.to_string())
}
```

// src-tauri/src/commands/mod.rs

## Error Handling in Commands

Tauri commands return `Result<T, E>` where `E` must implement `Serialize`.

```rust
// src-tauri/src/error.rs
use serde::Serialize;
use thiserror::Error;

// Commands must return a Serialize-able error
#[derive(Error, Debug, Serialize)]
#[serde(tag = "kind", content = "message")]
pub enum CommandError {
    #[error("not found: {0}")]
    NotFound(String),

    #[error("validation error: {0}")]
    Validation(String),

    #[error("internal error: {0}")]
    Internal(String),
}

// Convert from internal errors at the command boundary
impl From<anyhow::Error> for CommandError {
    fn from(err: anyhow::Error) -> Self {
        CommandError::Internal(err.to_string())
    }
}

// Use in commands — strongly typed errors serialized to frontend
#[tauri::command]
pub async fn get_user(
    id: String,
    state: State<'_, AppState>,
) -> Result<User, CommandError> {
    state.db
        .find_user(&id)
        .await?
        .ok_or_else(|| CommandError::NotFound(format!("user {id}")))
}
```

        // Register plugins

## Events (Rust → Frontend)

Push updates from Rust to the SvelteKit frontend.

```rust
// src-tauri/src/commands/sync.rs
use tauri::{AppHandle, Emitter};
use serde::Serialize;

#[derive(Clone, Serialize)]
pub struct ProgressPayload {
    pub current: u32,
    pub total: u32,
    pub message: String,
}

#[tauri::command]
pub async fn start_sync(app: AppHandle) -> Result<(), String> {
    // Emit progress events to all windows
    for i in 0..100 {
        app.emit("sync-progress", ProgressPayload {
            current: i,
            total: 100,
            message: format!("Syncing item {i}..."),
        })
        .map_err(|e| e.to_string())?;

        tokio::time::sleep(std::time::Duration::from_millis(50)).await;
    }
    Ok(())
}
```

## Frontend IPC (SvelteKit + TypeScript)

```typescript
// src/lib/tauri.ts — typed wrappers around invoke
import { invoke } from '@tauri-apps/api/core';

export interface User {
    id: string;
    name: string;
    email: string;
}

export interface CreateUserInput {
    name: string;
    email: string;
}

// Typed command wrappers — centralise all IPC here
export const commands = {
    greet: (name: string) =>
        invoke<string>('greet', { name }),

    getUsers: () =>
        invoke<User[]>('get_users'),

    createUser: (input: CreateUserInput) =>
        invoke<User>('create_user', { input }),

    getUser: (id: string) =>
        invoke<User>('get_user', { id }),

    startSync: () =>
        invoke<void>('start_sync'),
};
```

```svelte
<!-- src/routes/+page.svelte -->
<script lang="ts">
    import { onMount, onDestroy } from 'svelte';
    import { listen, type UnlistenFn } from '@tauri-apps/api/event';
    import { commands, type User } from '$lib/tauri';

    let users: User[] = [];
    let error: string | null = null;
    let syncProgress = 0;
    let unlisten: UnlistenFn;

    onMount(async () => {
        // Load data
        try {
            users = await commands.getUsers();
        } catch (e) {
            error = String(e);
        }

        // Listen for Rust events
        unlisten = await listen<{ current: number; total: number; message: string }>(
            'sync-progress',
            (event) => {
                syncProgress = Math.round((event.payload.current / event.payload.total) * 100);
            }
        );
    });

    onDestroy(() => {
        unlisten?.();
    });

    async function handleSync() {
        await commands.startSync();
    }
</script>

<ul>
    {#each users as user (user.id)}
        <li>{user.name} — {user.email}</li>
    {/each}
</ul>

{#if syncProgress > 0}
    <progress value={syncProgress} max={100}>{syncProgress}%</progress>
{/if}

<button onclick={handleSync}>Sync</button>
```

## Critical Rules (Tauri v2)

### Always

- Command errors must `impl Serialize` — use a typed `CommandError` enum, not raw strings.
- Keep heavy async work in Rust; the frontend only calls `invoke()` and renders.
- Clean up event listeners with `unlisten()` in `onDestroy` to prevent memory leaks.
- Grant only the minimum permissions needed in `capabilities/`.
- Use `AppHandle` (not `Window`) for emitting events — it works across all windows.

### Never

- Use `String` as the error type in commands (opaque, untyped for the frontend).
- Call `std::thread::sleep` in a Tauri command — use `tokio::time::sleep`.
- Block the main thread with `Mutex::lock()` across await points — use `tokio::sync::Mutex`.
- Emit events in a tight loop without backpressure — throttle or batch them.
- Put secrets in `tauri.conf.json` — it ships with the app binary.

### Ask First

- Adding new plugins (they require capability permission entries).
- Changing the app identifier (`com.example.myapp`) — affects OS-level data paths.
- Enabling mobile builds — requires additional platform setup.
