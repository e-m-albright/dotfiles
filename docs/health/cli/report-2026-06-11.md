## June-2026 best-practice research → adoption gaps

Full cited research is summarized in this run's chat and the run log. Verified
headlines and their bearing on this repo:

- **Converged standards:** `AGENTS.md` (Linux Foundation; 30+ tools) for passive
  rules, `SKILL.md` + `npx skills` for active capabilities. This repo already uses
  both. **Subagents, hooks, and permission profiles remain non-portable** — the
  CLI's per-vendor generator is the state-of-the-art workaround, not a solved
  standard. MCP is now under the LF Agentic AI Foundation; **A2A augments it**
  (agent coordination); nothing replaced it.
- **Sycophancy is empirically unsolved by prompting** (BrokenMath 2025: GPT-5 still
  29% sycophantic; "none eliminate it"). The durable lever is **mechanical
  verification gates**, not rules. This repo's "evidence before assertions" rule
  should graduate to a gate where the harness allows.
- **Code-health frontier:** LLM-augmented mutation testing (Meta ACH) and a
  GitClear-style **slop index** (clone growth, moved-vs-copied ratio) as
  *scheduled audits*, not gates.

### Controversial next steps (ranked, opinionated — for owner decision)

1. **Verify-before-done as a mechanical gate** (DO, highest ROI). A commit/PR check
   that a "done" claim carries attached command output. Best counter to sycophancy.
2. **Adversarial-reviewer pass with a fresh context** (DO). Formalize a mandatory
   diff critic; wire findings into the ratchet so they can't be ignored. (This pass
   used exactly that — a `code-reviewer` subagent found the 405 sibling bugs.)
3. **LLM-augmented mutation testing for the Python CLI** (DO, scoped). Scheduled,
   not pre-push. Measures whether tests catch injected bugs (the AI-slop defense).
4. **Slop-index scheduled audit** (DO as observability, not a gate). Watch the
   moved-vs-copied ratio on this repo's own history.
5. **Generate per-vendor subagent configs from one source** (DO cautiously, for
   subagents only). Hold on hooks — too vendor-divergent, low duplication today.
6. **A2A / homegrown MCP gateway** (DON'T — YAGNI). Enterprise governance gaps the
   foundation is solving; not a personal-repo problem.
