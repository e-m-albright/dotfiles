## What lives where in `~/.cursor/`

- `~/.cursor/cli-config.json` ŌĆö **dotfiles-managed** (symlink to `agents/cursor/cli-config.json`).
- `~/.cursor/mcp.json` ŌĆö **dotfiles-managed** (symlink to `editors/cursor/mcp.json`, generated by `agents/cursor/setup.sh`).
- `~/.cursor/rules/shared-rules.mdc` ŌĆö the kernel generated from `ai/agents/shared/rules.md`.
- `~/.cursor/plugins/dotfiles` ŌĆö symlink registering this directory as a local Cursor plugin.
- `~/.cursor/skills/` ŌĆö **dotfiles-managed** symlinks to canonical `ai/skills/`, plus any explicit local extras. Cursor reads this as the owned skill surface.
- `~/.cursor/skills-cursor/` ŌĆö **Cursor-vendor-shipped** built-in skills (`babysit`, `canvas`, `create-hook`, `create-rule`, etc.). Read-only, auto-managed by Cursor ŌĆö do **not** track in dotfiles.
- Everything else (`projects/`, `chats/`, `plans/`, `ai-tracking/`, `statsig-cache.json`, ŌĆ”) is local state and intentionally gitignored.

## Install Notes

- Marketplace plugins can bundle skills, agents, hooks, rules, commands, and MCP servers.
- MCP servers from plugins are toggled under Cursor Settings > Features > Model Context Protocol.
- For Parallel setup, run `/parallel-setup` after install (requires `parallel-cli` auth).
- For profile-specific behavior, use dotfiles-managed MCPs and keep marketplace plugin installs per machine/account.

## Install Commands

- `/add-plugin parallel` (optional)
- `/add-plugin cursor-team-kit` (optional)
- `/add-plugin tldraw` (optional)
