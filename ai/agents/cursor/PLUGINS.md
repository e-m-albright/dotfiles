## What lives where in `~/.cursor/`

- `~/.cursor/cli-config.json` — **dotfiles-managed** (symlink to `agents/cursor/cli-config.json`).
- `~/.cursor/mcp.json` — **dotfiles-managed** (symlink to `editors/cursor/mcp.json`, generated by `agents/cursor/setup.sh`).
- `~/.cursor/rules/shared-rules.mdc` — the kernel generated from `ai/agents/shared/rules.md`.
- `~/.cursor/plugins/dotfiles` — symlink registering this directory as a local Cursor plugin.
- `~/.cursor/skills/` — skills installed by **plugins or via `/add-skill`**; tracked elsewhere via `ai/skills/` (deployed by Claude/Codex). Cursor reads them but we don't author them here.
- `~/.cursor/skills-cursor/` — **Cursor-vendor-shipped** built-in skills (`babysit`, `canvas`, `create-hook`, `create-rule`, etc.). Read-only, auto-managed by Cursor — do **not** track in dotfiles.
- Everything else (`projects/`, `chats/`, `plans/`, `ai-tracking/`, `statsig-cache.json`, …) is local state and intentionally gitignored.

## Install Notes

- Marketplace plugins can bundle skills, agents, hooks, rules, commands, and MCP servers.
- MCP servers from plugins are toggled under Cursor Settings > Features > Model Context Protocol.
- For Parallel setup, run `/parallel-setup` after install (requires `parallel-cli` auth).
- For profile-specific behavior, use dotfiles-managed MCPs and keep marketplace plugin installs per machine/account.

## Install Commands

- `/add-plugin superpowers`
- `/add-plugin context7-plugin`
- `/add-plugin parallel` (optional)
- `/add-plugin cursor-team-kit` (optional)
