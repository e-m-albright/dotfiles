# Verification-claim patterns for the verify-before-done Stop hook (kernel K1).
# One extended-regex (grep -iEf) per line; '#' lines and blanks are ignored.
# These are STRONG "I checked and it's good" claims — the hook only nudges when
# one of these appears AND the agent ran zero tools that turn. Tune freely:
# add a pattern the moment a lie slips through; loosen one that false-positives.
tests? (now )?(all )?(pass|passing|passes|passed|are green)
all (the )?tests? (are )?(now )?(passing|green)
all (tests? )?green
ci('?s| is) (now )?green
build (now )?(succeeds|passes|passed|is green)
it (now )?works( now)?
verified (it|that|this)?\s*works
confirmed (it|this|that)?\s*works
everything (passes|works|is green|is passing)
all (checks?|gates?) (pass|passing|green)
successfully (tested|verified|ran)
the (fix|change|build) (works|is verified|passed)
